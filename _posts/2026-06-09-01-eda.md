# EDA — House Prices Prediction

Kaggle의 [House Prices: Advanced Regression Techniques](https://www.kaggle.com/c/house-prices-advanced-regression-techniques) 데이터를 가지고 집값을 예측해보는 첫 번째 ML 프로젝트입니다. 일단 이번 단계에서는 데이터가 어떻게 생겼는지 파악하는 것부터 시작했습니다.

---

## 데이터 구조 파악

```python
Train shape: (1460, 81)
Test shape:  (1459, 80)
```

컬럼이 81개나 됩니다. 처음에는 이 많은 컬럼을 어떻게 다 다뤄야 하나 싶었는데, 일단 `train.info()`로 타입부터 확인해봤습니다.

int64 35개, float64 3개, object 43개로 구성되어 있었습니다. 수치형 컬럼보다 object 타입이 더 많다는 게 의외였습니다. 찾아보니 ExterQual(외장 품질), KitchenQual(주방 품질), BsmtCond(지하실 상태) 같은 등급 컬럼들이 전부 문자열로 저장된 것이었습니다. "Ex(Excellent)", "Gd(Good)", "TA(Average)" 이런 식으로요. 나중에 모델에 넣으려면 숫자로 바꿔야 할 것 같습니다.

float64가 딱 3개뿐인 것도 의아했는데, LotFrontage, MasVnrArea, GarageYrBlt를 보니 셋 다 결측값이 있는 컬럼이었습니다. 나중에 알게 됐지만 pandas에서 정수 컬럼에 NaN이 생기면 자동으로 float으로 바뀐다고 합니다.

## SalePrice 분포 확인

타겟 변수인 SalePrice부터 어떻게 생겼는지 살펴봤습니다.

```python
train["SalePrice"].describe()
```

| 통계량 | 값 |
|--------|-------------|

평균이 $180,921인데 중앙값이 $163,000입니다. 평균이 중앙값보다 꽤 높길래 히스토그램으로 그려봤습니다.

![saleprice_distribution.png](/assets/img/posts/2026-06-09-01-eda/saleprice_distribution.png)

역시나 오른쪽으로 꼬리가 길게 늘어져 있었습니다. 집값이 $34,900에서 $755,000까지 분포하는데, 비싼 집 몇 채가 분포를 오른쪽으로 잡아당기고 있는 모습입니다. 검색해보니 이런 분포에서는 로그 변환을 많이 쓴다고 해서, 전처리 단계에서 `np.log1p(SalePrice)`를 적용해볼 생각입니다.

## 결측치 파악

결측이 있는 컬럼들을 비율 순으로 정렬해봤습니다.

```python
missing_ratio = (train.isnull().sum() / len(train)) * 100
missing_ratio[missing_ratio > 0].sort_values(ascending=False)
```

| 컬럼 | 결측 수 | 결측 비율 |
|------|--------:|----------:|
| PoolQC | 1,453 | 99.52% |
| MiscFeature | 1,406 | 96.30% |
| Alley | 1,369 | 93.77% |
| Fence | 1,179 | 80.75% |
| MasVnrType | 872 | 59.73% |

| GarageType | 81 | 5.55% |

| BsmtCond | 37 | 2.53% |
| BsmtFinType1 | 37 | 2.53% |
| MasVnrArea | 8 | 0.55% |
| Electrical | 1 | 0.07% |

PoolQC가 99.52% 결측인 걸 보고 처음에는 데이터에 문제가 있나 싶었습니다. 그런데 data_description.txt를 읽어보니 PoolQC는 수영장 품질을 나타내는 컬럼이었고, 수영장이 없는 집은 자연히 이 값이 없는 것이었습니다. Alley(골목 접근 방식), Fence(울타리 유형), FireplaceQu(벽난로 품질)도 같은 이유였습니다. 결측이 "데이터 누락"이 아니라 "해당 시설 없음"을 의미하는 것들이었습니다.

차고 관련 컬럼들(GarageType, GarageYrBlt, GarageFinish, GarageQual, GarageCond)이 전부 똑같이 81개 결측인 것도 같은 맥락인 것 같습니다. 차고가 없는 집 81채가 그대로 결측으로 남은 것으로 보입니다. 지하실 컬럼들도 37~38개로 거의 비슷합니다.

반면 `LotFrontage`는 17.74%(259개)가 결측인데, 이건 도로와 접한 길이를 기록하지 못한 것이라 단순히 "없음"으로 채울 수 없습니다. 같은 Neighborhood끼리는 lot 크기가 비슷할 것 같아서 Neighborhood별로 중앙값을 구해서 채워볼 계획입니다.

`MasVnrType`(석조 외장 종류)은 결측이 872개나 되는데 `MasVnrArea`(석조 외장 면적)는 겨우 8개인 것도 이상했습니다. 확인해보니 면적이 0인 집들은 타입 자체가 없어서 결측으로 남은 경우가 대부분이었습니다. 이것도 "없음" 처리가 맞을 것 같습니다.
